How does javascript work?

JavaScript is a single threaded client side programming language. Single threaded means, it can do a work at a time. JavaScript code runs on web browser and also terminal with the help of nodejs run time. But how can web browser and nodejs understand the javascript code? Every web browser's has a mechanism or a engine. Example - chrome browser has v8 engine, edge browser has chakra engine, safari browser has javascript core engine, firebox browser has spidermonkey engine etc. The browser engine then converts the JavaScript code into machine language. Then our machine can understand the code and display the output. Also nodejs use v8 engine.

What does javascript event loop do?

By default JavaScript works synchronously. What does mean synchronous? This means it will block all other work until it is finished. If there is a heavy work, then it will take long time. For this reason, we can not do other work until the work will finish. Javascript can also work on asynchronously. If javascript do heavy work asynchronously then it will not block other works. Browser engine has two parts, HEAP and CALL STACK and also some assistant named WEB APIS and CALLBACK QUEUE. CALL STACK executes all our JavaScript code. If CALL STACK found any asynchronous work, then it transfer this work to the web api. Web api after finish the work, it transfer the work to the CALLBACK QUEUE. When the CALL STACK is empty, there is a middle person called Event Loop, who transfers the work from the CALLBACK QUEUE to the CALL STACK. Then the CALL STACK executes the work.
